URL Format

Dissecting the URL

Now let's dissect the components of the URL to understand what is happening:

ComponentDescriptionExample
Base URLBase URL for the APIhttps://www.nwfsc.noaa.gov/data/api/v1/source
LayerName of the layer you're querying.trawl.catch_fact
ContentThe type of content you want to receive. selection.json
Query FiltersStarts with filters= then appended with a listing of comma separated key=value pairsfilters=field_identified_taxonomy_dim$scientific_name=Eopsetta%20jordani,date_dim$year>=2010,date_dim$year<=2012
Requested VariablesList of variables, comma separatedvariables=date_yyyymmdd,field_identified_taxonomy_dim$scientific_name
Complete URL
https://www.nwfsc.noaa.gov/data/api/v1/source/trawl.catch_fact/selection.json?filters=field_identified_taxonomy_dim$scientific_name=Eopsetta%20jordani,date_dim$year%3E=2010,date_dim$year%3C=2012&variables=date_yyyymmdd,field_identified_taxonomy_dim$scientific_name

It is important to note that the content type is followed by an ? symbol and between the filters and variables is an & symbol. Now it is simply a matter of identifying which layer you want to query, what are your query filters, and which variables you want to return from that query.
